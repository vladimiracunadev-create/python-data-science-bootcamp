# 🛠️ Tecnologías complementarias — Clase 00: Diagnóstico inicial y orientación

> Herramientas y librerías que complementan o extienden lo visto en esta clase.

## 📦 Librerías principales usadas en esta clase

| Librería | Versión recomendada | Para qué se usa en esta clase |
|---|---|---|
| Python | 3.10+ | Lenguaje base del bootcamp; se evalúa conocimiento previo en el diagnóstico |
| Jupyter | 7.x | Entorno interactivo donde se trabajarán todos los notebooks del bootcamp |
| pandas | 2.x | Librería central para análisis de datos; referenciada en el diagnóstico como área clave |

## 🔗 Herramientas complementarias

| Herramienta | Descripción | Cuándo usarla |
|---|---|---|
| Google Forms / Typeform | Formularios digitales para recolectar respuestas del diagnóstico | Cuando se aplica el diagnóstico en formato digital antes de la clase |
| Google Sheets / Excel | Hoja de cálculo para registrar y filtrar los resultados del grupo | Para organizar los resultados antes de pasarlos a Python |
| Notion / Markdown | Documentación del plan de ruta personalizado por estudiante | Para registrar el acuerdo de aprendizaje al cierre de la sesión |
| VS Code | Editor de código con soporte para notebooks y Python | Alternativa a Jupyter para quienes prefieren un entorno de escritorio completo |

## 📚 Recursos para profundizar

- [Documentación oficial de Python](https://docs.python.org/es/3/): documentación oficial en español para repasar conceptos básicos detectados en el diagnóstico
- [Jupyter — Primeros pasos](https://jupyter-notebook.readthedocs.io/en/stable/): recurso para profundizar en el uso del entorno de trabajo del bootcamp
- [Roadmap de Data Science — Kaggle Learn](https://www.kaggle.com/learn): guía práctica con rutas estructuradas para los temas que cubre el bootcamp
