# Herramientas pedagogicas de aula

## Objetivo

Tener una caja de herramientas concreta para mostrar en entrevista o usar en clase de prueba. El foco no es solo decir que sabes ensenar, sino mostrar como lo haces.

## Estructura base de una sesion

1. Apertura con objetivo visible.
2. Activacion breve.
3. Demostracion guiada.
4. Practica acompanada.
5. Chequeo de comprension.
6. Cierre con interpretacion.

## Herramientas para abrir bien una clase

- decir que van a aprender y para que sirve;
- activar con una pregunta corta;
- mostrar el resultado final esperado;
- bajar la ansiedad con una primera tarea facil.

## Herramientas para explicar

- usar una sola idea fuerte por bloque;
- pasar de ejemplo pequeno a aplicacion concreta;
- verbalizar lo que haces y por que;
- pedir prediccion antes de ejecutar.

## Herramientas para practicar

- ejercicio base obligatorio;
- desafio opcional para quienes avanzan rapido;
- parejas o mini grupos cuando hay miedo a equivocarse;
- checkpoints cortos cada 15 a 20 minutos.

## Herramientas para verificar aprendizaje

- pedir que expliquen una linea de codigo;
- cambiar una variable o condicion y pedir ajuste;
- pedir una conclusion escrita en lenguaje simple;
- usar un mini ticket de salida.

## Herramientas para manejar diferencias de ritmo

- minimo comun claramente definido;
- apoyo guiado para quienes se bloquean;
- reto adicional para quienes terminan antes;
- revision colectiva de un error comun.

## Herramientas para manejar cualquier tecnologia en clase

El punto no es pelear contra la tecnologia. Es ordenar su uso.

Regla sugerida:

1. pensar;
2. consultar;
3. verificar;
4. explicar.

## Aplicacion concreta

- si usan buscadores o asistentes, primero formulan una hipotesis;
- si reciben una respuesta, la contrastan con el objetivo;
- si copian codigo, deben adaptarlo y explicarlo;
- si la herramienta propone algo mas complejo que la clase, se reduce al nivel del grupo.

## Problemas frecuentes y respuesta docente

| Problema | Respuesta recomendada |
|---|---|
| miedo a equivocarse | victoria rapida + preguntas de baja exposicion |
| copia sin comprension | pedir explicacion y adaptacion |
| frustracion por errores | normalizar bug + rutina de lectura del error |
| ritmos muy distintos | minimo comun + desafio opcional |
| baja participacion | practica en parejas + cierres breves |
| dependencia excesiva de tecnologia | volver al objetivo y pedir justificacion |

## Indicadores de una buena clase

- el estudiante entiende lo que hizo;
- puede explicar una variacion pequena;
- conecta el resultado con una conclusion;
- no depende solo de copiar pasos.

## Frase breve para entrevista

"Mi foco pedagogico no es solo que ejecuten codigo. Es que comprendan la logica, puedan verificar resultados y ganen confianza para reutilizar lo aprendido."
