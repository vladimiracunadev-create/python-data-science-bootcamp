# 🖼 Bootcamp Python para Data Science

## Demo de mi sistema de capacitación

### Adaptado para Skillnest / Colegio San Nicolas de Maipu

Entrevista: jueves 9 de abril de 2026, 15:30

---

## 1. Qué traigo hoy

- Un repositorio docente ya funcional.
- Una muestra concreta de como diseño capacitaciones.
- Una propuesta V1 aterrizada al colegio.
- Una base que puede crecer aunque esta oportunidad no avance.

---

## 2. Qué demuestra este demo sobre mi forma de trabajar

- Trabajo con progresión pedagógica, no con contenido suelto.
- Conecto explicación, práctica, evaluación y cierre.
- Puedo adaptar el nivel sin perder estructura.
- Ya tengo una base reutilizable para distintas implementaciónes.

---

## 3. Por que esto sigue teniendo valor aunque existan nuevas tecnologias

- La tecnología puede asistir, pero no secuenciar aprendizaje por si sola.
- No detecta confusion, copia sin comprensión o miedo a equivocarse.
- No convierte una respuesta en criterio, interpretacion y confianza.
- Mi valor esta en la mediación pedagógica y en el diseño de la experiencia.

---

## 4. Qué ya resuelve este repositorio

- Clase 0 diagnóstica + 12 clases de 90 minutos con objetivos, notebooks, ejercicios y soluciones.
- Datasets sinteticos listos para usar.
- Documentación de metodología, evaluación y guía docente.
- App local para practicar Python dentro del mismo proyecto.
- Landing pública para alumnos en GitHub Pages y ruta clara de evolución a móvil.

---

## 5. Cómo lo acotaría para una primera versión escolar

- Foco en Python, pandas y visualización básica.
- Una sola ruta principal de aprendizaje.
- Práctica guiada antes que complejidad técnica.
- Machine Learning y pipelines quedan para fase 2.

---

## 6. Ruta V1 recomendada

1. Fundamentos de Python aplicados a datos.
2. Lectura de CSV con pandas.
3. Filtros, columnas y tablas resumen.
4. Gráficos simples e interpretacion.
5. Mini proyecto guiado.
6. Presentación breve de hallazgos.

---

## 7. Cómo trabajaría en sala y que problemas tengo previstos

- Bloques de 90 minutos.
- Explicación breve.
- Demostracion guiada.
- Ejercicio individual o en parejas.
- Cierre con retroalimentación.
- Manejo de ritmos distintos, errores de sintaxis y uso de IA con reglas claras.

---

## 8. Cómo crece esta base

- De demo breve a piloto acotado.
- De piloto a bootcamp completo.
- De colegio a empresa o programa social con la misma base.
- De muestra técnica a sistema de capacitación reusable.

---

## 9. Qué necesito confirmar para ejecutar bien

- Horas reales por semana.
- Fecha de inicio efectiva.
- Nivel y cantidad de estudiantes.
- Equipamiento disponible.
- Alcance pagado de preparación, evaluación y adaptación.

---

## 10. Mi valor para este proyecto

- Puedo traducir contenido técnico a aprendizaje claro.
- Ya tengo una base docente real, no una idea en abstracto.
- Puedo adaptar el material con criterio sin improvisar.
- Priorizo una primera implementación ejecutable, no una promesa inflada.
