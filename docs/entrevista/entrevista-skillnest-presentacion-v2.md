# Presentacion breve para entrevista

## Punto de partida

- Repositorio docente ya funcional.
- Muestra concreta de como diseno capacitaciones.
- Propuesta V1 aterrizada al colegio.
- Base reusable que puede crecer con o sin esta oportunidad.

## Que demuestra esta muestra

- trabajo con progresion pedagogica, no con contenido suelto;
- conecto explicacion, practica, evaluacion y cierre;
- puedo adaptar el nivel sin perder estructura;
- ya existe una base reusable para distintas implementaciones.

## Por que sigue teniendo valor aunque existan nuevas tecnologias

- la tecnologia puede asistir, pero no secuenciar aprendizaje por si sola;
- no detecta confusion, copia sin comprension o miedo a equivocarse;
- no convierte una respuesta en criterio, interpretacion y confianza;
- el valor esta en la mediacion pedagogica y en el diseno de la experiencia.

## Que ya resuelve este repositorio

- clase 0 diagnostica y 12 clases de 90 minutos con objetivos, notebooks, ejercicios y soluciones;
- datasets sinteticos listos para usar;
- documentacion de metodologia, evaluacion y guia docente;
- app local para practicar Python dentro del mismo proyecto;
- portal publico para alumnos y una ruta clara de evolucion a movil.

## Como lo acotaria para una primera version escolar

- foco en Python, pandas y visualizacion basica;
- una sola ruta principal de aprendizaje;
- practica guiada antes que complejidad tecnica;
- machine learning y pipelines quedan para una segunda etapa.

## Ruta V1 recomendada

1. Fundamentos de Python aplicados a datos.
2. Lectura de CSV con pandas.
3. Filtros, columnas y tablas resumen.
4. Graficos simples e interpretacion.
5. Mini proyecto guiado.
6. Presentacion breve de hallazgos.

## Que necesito confirmar para ejecutar bien

- horas reales por semana;
- fecha de inicio efectiva;
- nivel y cantidad de estudiantes;
- equipamiento disponible;
- alcance pagado de preparacion, evaluacion y adaptacion.

## Cierre sugerido

Puedo traducir contenido tecnico a aprendizaje claro, ya tengo una base docente real y puedo adaptar el material con criterio sin improvisar. Mi prioridad no es vender una promesa inflada, sino proponer una primera implementacion ejecutable.
