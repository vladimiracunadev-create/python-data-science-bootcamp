# 🎬 Demo sugerida para la reunión

## Objetivo de la demo

Mostrar que el repositorio ya esta pensado como una base docente real, no solo como código suelto.

## Recorrido recomendado (5 a 7 minutos)

1. **README.md**
   - Explicar que el repositorio combina planificacion, clases y práctica.
2. **docs/syllabus.md**
   - Mostrar la progresión del bootcamp y la lógica pedagógica.
3. **classes/01-python-fundamentos/**
   - Abrir el notebook y comentar la estructura.
4. **classes/02-pandas-limpieza-datos/**
   - Mostrar continuidad entre fundamentos y análisis aplicado.
5. **app/**
   - Ejecutar `python -m app.app` o `python run_bootcamp.py`.
   - Mostrar vista de clases, notebook por celdas y runner.
6. **mini_project/desafío.md**
   - Cerrar mostrando evaluación y aplicacion real.

## Mensaje sugerido

> Prepare este repositorio como muestra de como abordaria un bootcamp: con estructura de clases, notebooks, ejercicios, evaluación y un entorno local para practicar dentro del mismo proyecto. Quise dejarlo simple de usar para estudiantes, pero suficientemente ordenado para reutilizarlo como base docente real.

## Puntos que más te conviene remarcar

- experiencia previa en relatorias y entornos educativos;
- capacidad de traducir lo técnico a aprendizaje practico;
- material reutilizable y adaptable por nivel;
- entorno local ejecutable sin depender obligatoriamente de Docker.
