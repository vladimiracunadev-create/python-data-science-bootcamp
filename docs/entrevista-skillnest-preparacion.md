## Preguntas que probablemente te haran

## "Como adaptarias esto a estudiantes de colegio?"

Respuesta corta:

- partiria con Python, pandas y visualizacion;
- trabajaria con bloques de 90 minutos;
- priorizaria practica guiada y ejercicios simples;
- dejaria ML para una segunda etapa.

## "Como manejas grupos con niveles distintos?"

Respuesta corta:

- objetivo minimo comun para todo el grupo;
- ejercicio base obligatorio;
- desafio opcional para quienes avanzan mas rapido;
- cierre colectivo para consolidar.

## "Por que este repositorio te representa bien como docente?"

Respuesta corta:

- porque muestra secuencia, criterio pedagogico y ejecucion;
- no es solo teoria ni solo codigo;
- demuestra como transformo contenido tecnico en una experiencia de aprendizaje usable.

## "Que harias en la primera semana?"

Respuesta corta:

- nivelacion breve;
- introduccion a Python;
- primer contacto con CSV y pandas;
- una victoria rapida para que el grupo no pierda confianza.

## "Que necesitarias para ejecutar bien el programa?"

Respuesta corta:

- perfil del grupo;
- cantidad de estudiantes;
- horario real confirmado;
- equipamiento disponible;
- definicion de evaluacion, pago y alcance.

## Limites sanos para que no se aprovechen

- Puedes mostrar una muestra concreta y bien pensada.
- No necesitas entregar una planificacion completa a medida antes de acordar condiciones.
- No necesitas producir varios PDFs, evaluaciones, rubricas y una clase demo larga sin definir si es parte pagada del proceso.
- Si te piden una clase de prueba extensa o una adaptacion curricular completa, conviene mover la conversacion a una etapa pagada o acotada por escrito.

## Frases utiles para poner limite con buen tono

"Puedo mostrar una muestra breve y bien adaptada para que vean mi enfoque. Si quieren una version ya cerrada para implementacion, feliz la preparo una vez definido el alcance y la contratacion."

"Tengo disponibilidad para ajustar el material, pero prefiero hacerlo sobre una base confirmada de horas, grupo y objetivos para no construir algo que despues no coincida con la realidad del curso."

"Si esperan una clase de prueba o una personalizacion mayor, me sirve dejar claro desde ya si eso forma parte de la evaluacion del proceso o de una etapa ya pagada de preparacion."

## Senales de alerta

- Piden demasiado material especifico antes de hablar de pago.
- Hablan de horas de clase, pero no de preparacion, traslado y coordinacion.
- Quieren quedarse con el material completo aunque no haya acuerdo.
- Cambian la carga horaria o el alcance varias veces sin formalizar nada.
- Te piden resolverles el diseno del programa antes de contratar.
